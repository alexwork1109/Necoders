## 🏛 О проекте: Полная цифровизация бюджетного процесса

Органы власти Амурской области достигли **полной цифровизации бюджетного процесса**. Данные об объектах и мероприятиях государственных программ хранятся в различных информационных системах с определённым набором показателей, связанных между собой сквозной кодировкой.

### Суть проблемы
Несмотря на высокий уровень оцифровки, аналитики ежедневно сталкиваются с **разрозненностью информации**. Данные лежат в различных тяжеловесных `.csv` выгрузках (бюджетная роспись, соглашения, государственные задания), требуя долгих часов ручной сверки по сквозным кодам для формирования единой и прозрачной картины. 

### Наше решение
Мы разработали **умный конструктор выборок** — единый пункт управления бюджетными данными. Платформа самостоятельно собирает сырые выгрузки из разрозненных систем, автоматически связывает их с помощью строгих алгоритмов ETL и формирует реляционную структуру данных. Это позволяет аналитику формировать срезы и получать интерактивные дашборды «на лету», принимая управленческие решения за секунды.

## 🔥 Ключевые возможности

*   🧠 **Smart Data Ingestion (Умный импорт):** Наш ETL-конвейер автоматически распознает типы загружаемых реестров, нормализует форматы дат, сумм и интеллектуально связывает данные об объектах по сквозной кодировке.
*   ⚡ **Ultra-Fast Search (Ультрабыстрый поиск):** Оптимизированная база данных обеспечивает поиск нужных инфраструктурных объектов и мероприятий за **< 50 мс**.
*   📊 **Интерактивная аналитика:** Продвинутые React-компоненты динамически визуализируют кассовое исполнение, позволяя сопоставлять плановые назначения с фактическими расходами.
*   🚀 **Enterprise Готовность:** Система полностью автономна. Поддерживает работу в **изолированных закрытых контурах (On-Premise)** без доступа к сети Интернет, сертифицирована и оттестирована для запуска на **РЕД ОС**.
*   📑 **Конструктор отчетов:** Мгновенная генерация и экспорт любых сформированных аналитических срезов в сводные `Excel` файлы для беспрепятственного доклада руководству.

## 🏗 Архитектура и Технологический стек

Продукт спроектирован на базе современной (container-ready) архитектуры, что обеспечивает абсолютную отказоустойчивость, безопасность хранения данных и возможность бесконечного горизонтального масштабирования.

### 💻 Frontend (Пользовательский интерфейс)
Обеспечивает реактивность и мгновенный отклик при работе со сложными пользовательскими выборками.
*   **React 18** + **TypeScript** — строгая типизация, безопасность и предсказуемость UI.
*   **Vite** — сверхбыстрая сборка и максимальная производительность в браузере.
*   **Custom UI Kit / Tailwind** — строгий, эстетичный и премиальный корпоративный интерфейс.

### ⚙️ Backend (Ядро агрегации)
Гарантирует обработку гигантских массивов сырых данных и моментальную выдачу готовых агрегированных срезов через REST API.
*   **Python 3.12** — максимальная производительность парсинга.
*   **Flask** — гибкий, быстрый и надежный веб-фреймворк.
*   **Pydantic** — глубокая валидация и очистка входящих данных.
*   **SQLAlchemy** — мощнейшая ORM для безопасной транзакционной работы с СУБД.
*   **Gunicorn** — production-ready сервер для многопоточной Enterprise-обработки запросов.

### 🐳 Инфраструктура
*   **Docker & Docker Compose** — стопроцентная изоляция окружения.
*   **Amvera Cloud** — DevOps манифесты (`amvera.yaml`) для облачного CI/CD деплоя.
*   **Кроссплатформенность** — нативная поддержка импортозамещенных систем.

## 🚀 Развертывание платформы (Deployment)

Модель поставки системы позволяет произвести интеграцию за пару минут как на локальном АРМ разработчика, так и загрузить её на удаленный боевой север администрации.

### Вариант 1: Быстрый старт (Локально: Windows)
Все зависимости установятся автоматически, база данных развернется из коробки.
```powershell
prepare.cmd  # Инициализация окружения (выполнить один раз)
dev.cmd      # Поднятие серверов разработки Frontend + API
```
*Интерфейс:* `http://localhost:5173` | *API:* `http://localhost:5000`

### Вариант 2: Серверный запуск (Linux / macOS / РЕД ОС)
Гладкий запуск для классических UNIX систем.
```bash
chmod +x ./prepare.sh ./dev.sh ./clean.sh
./prepare.sh
./dev.sh
```

### Вариант 3: Production контур (Docker-контейнеризация)
Рекомендованный способ для показа жюри или окончательного внедрения. Всё изолировано и безопасно работает "под капотом".
```bash
# 1. Multi-stage сборка 
docker build -t constructor-analytics .

# 2. Запуск ядра платформы (с маппингом хранилища данных)
docker run -d --name amur_analytics \
  -p 8080:8080 \
  -v $(pwd)/data:/data \
  --restart unless-stopped \
  constructor-analytics
```
*Боевая инсталляция доступна по адресу:* `http://localhost:8080`
